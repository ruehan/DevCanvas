## 배경
ADR-0007 이 이월한 재시도 정책을 실 GLM 관측으로 확정. 관측 하네스(`api/scripts/glm_observe.py`)로
z.ai `glm-4.7-flash`(무료) 호출 → requirement 9/9(100%), edit 0/3(전부 429/timeout, 스키마 위반 0).
결론: 전송 계층 지수 백오프 재시도(ADR-0021), 스키마 재시도는 근거 없어 보류.

## 1라운드 (code-reviewer, api 스코프)
- 판정: 수정 필요(경미)
- 검증: verify-all.sh EXIT 0 — api(ruff/mypy strict/pytest), web(변경 없음).
- 지적사항:
  - [P2-1] `_fetch_content` 의 도달불가 최종 `raise` 가 `max_retries < 0` 일 때 `last_exc=None`
    노출("GLM 호출 실패: None"). __init__ 에서 max_retries >= 0 보장 필요.
- 반영: `self._max_retries = max(0, raw_retries)` 클램프(음수 = 재시도 없음). 도달불가 분기는
  타입 완결성용으로 유지.
- 확인(리뷰어): 재시도 횟수(= max_retries+1), transient 분류(KeyError 는 비재시도 정상),
  백오프 수식(base*2**attempt), 무한루프/오류 삼킴 없음.

### 이월(non-blocking)
- edit 경로 프롬프트 비대(`_build_prompt` 전체 스키마+덤프) → 무료 티어 429 근본 원인.
  재시도만으로 실 edit 통과 못 함(재관측 확인). 프롬프트 축약/캐싱은 편집 사이클 별도 ADR(ADR-0021 영향에 명시).

## 검증 상태(정직)
- 재시도·백오프·타임아웃 로직: **단위 테스트 검증**(주입 timeout/429 시퀀스, sleep 주입으로 무대기).
- 무료 티어 실 edit 통과: **미달성** — 프롬프트 축약(후속) 전제. ADR-0021 에 한계 명시.
