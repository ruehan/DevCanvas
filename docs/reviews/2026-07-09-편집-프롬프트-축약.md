# 리뷰 기록 — 편집 프롬프트 축약(스키마 생략)

- 날짜: 2026-07-09
- 브랜치: feat/edit-prompt-shrink
- 최종 판정: 통과 (1라운드, 지적 0)

## 배경
ADR-0021 이 지목한 실 blocker: 편집 턴 프롬프트 비대. 측정상 formal JSON 스키마가 payload ~68%.
편집은 `current_result`(완전한 인스턴스)를 이미 컨텍스트로 받으므로 스키마 중복 → 생략(ADR-0022).

## 1라운드 (code-reviewer, api 스코프)
- 판정: **통과 (지적 0)**
- 검증: verify-all.sh EXIT 0 — api(ruff/mypy strict/pytest, 테스트 2종 추가/갱신), web(변경 없음).
- 확인(리뷰어): include_schema=False 분기가 유효 파싱 가능 프롬프트 생성, 포트 시그니처
  하위호환(requirement 등은 기본 True 유지), keyword-only 인자 3개 구현체 일관.
- 측정: 프롬프트 67% 축소(8539자→2804자, ~2846→934 tok).

## 실 관측(정직)
- 축소 후에도 무료 티어 실 edit = 429 지속. 격리(쿨다운+단건+재시도)에서도 429.
  → 연타 아티팩트/프롬프트 크기 아님, **무료 티어 쿼터 벽**. requirement 는 계속 100%.
- 성과 = 프롬프트 67% 축소(비용·지연·쿼터압박 감소). **무료 티어 edit 통과는 미달성**
  (유료 티어/쿼터 리셋 전제 — 후속). 로직은 단위 테스트 검증.
