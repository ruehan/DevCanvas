# 0002. MVP 범위를 관리자/대시보드/내부툴로 좁힘
- 상태: 채택
- 날짜: 2026-07-01

## 배경
기획안은 랜딩/로그인/온보딩/결제 등 광범위 화면 유형을 다룬다. 그러나 동시에 여러 유형을
지원하면 각 유형별 품질이 떨어지고, 특히 "구현 가능한 코드"라는 차별화 가치가 희석된다.

## 결정
MVP 화면 유형을 **관리자 페이지 / SaaS 대시보드 / 내부툴 UI** 세 가지로 한정한다.

## 이유와 대안
- 이 영역은 개발자가 반복적으로 만들고, 디자인 리소스가 부족하며, 생성 코드의 실사용 가치가 가장 큼(기획안 19장).
- 테이블/필터/상태 배지/차트 등 패턴이 정형화되어 품질 검증이 쉽다.
- 대안(넓은 범위): 빠른 어필은 가능하나 품질 분산 → "또 하나의 AI 디자인 툴"로 동질화.

## 영향
- 랜딩/로그인/온보딩은 Phase 2로 연기.
- 입력 폼의 "화면 유형" 선택지가 MVP에서 3개로 제한.
