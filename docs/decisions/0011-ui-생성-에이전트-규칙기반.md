# 0011. UI 생성 에이전트는 규칙 기반
- 상태: 채택
- 날짜: 2026-07-01

## 배경
ui_generator_agent 는 화면별 레이아웃(공간 배치)과 컴포넌트 트리(렌더 순서)를 생성한다.
design_system(0008)/ux_planner(0010)과 같은 근거로 규칙 기반이 적합하다:
- 입력이 이미 정형(UXPlan 의 ScreenSpec + ScreenKind)이라 배치는 종류별 템플릿으로 결정 가능.
- 결정적이어야 code_generator(다음 단계)가 일관된 입력을 받아 코드를 생성.
- LLM 의 자유도는 이 단계에서 불필요(오히려 일관성 해침).

## 결정
- ui_generator_agent 는 **규칙 기반**으로 동작. `pipeline/ui/generator.py` 의
  `build_ui_generation(ux_plan)` 이 각 화면의 ScreenKind 에서 ScreenLayout 을 도출한다.
- `pipeline/ui/templates.py` 에 종류별 layout(공간 배치 문자열)·component_tree(렌더 순서) 템플릿.
- LLM 정제(브랜드 맞춤 레이아웃 변형)는 향후 확장. 현재 llm.generate 를 사용하지 않는다.

## 이유와 대안
- 규칙 기반: code_generator 와의 결정적 계약, 종류별 일관, 테스트 용이.
- 대안(LLM 기반): 유연하나 비결정성·일관성 위험. MVP 에서 기각.

## 영향
- ui_generator_agent 가 LLM 포트를 사용하지 않음 → 규칙 기반 파이프라인 구간 추가.
- UIGeneration.layouts 은 UXPlan.screens 와 1:1. 종류가 같으면 동일 배치(의도적).
- **정합 규칙**: component_tree 는 top-level 영역 컴포넌트만 나열하며, 상위 단계
  `screen.components`(ux/templates)의 **부분집합**이어야 한다. layout 설명 문자열의
  각 영역 토큰도 ux/ui 컴포넌트명과 일관되게 작성한다(존재하지 않는 가짜 컴포넌트명
  금지). → code_generator 가 컴포넌트 정의 출처를 screen.components 에서 일관되게
  찾을 수 있다. runtime 부분집합 단정(test)으로 회귀를 잡는다.
