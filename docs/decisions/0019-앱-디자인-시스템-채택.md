# 0019. DevCanvas 앱 디자인 시스템 채택 (warm neutral · design.html 기반)
- 상태: 채택
- 날짜: 2026-07-09

## 배경
design-brief(v2)가 지시한 "Claude Design 류 감성"을 구체 디자인으로 구현한 `design.html`을
진실 원천으로 채택. 이 파일이 곧 앱 자체 UI 의 디자인 시스템(색·타이포·애니메이션·질감)이다.

> 주의: DevCanvas 가 *생성하는 결과물*의 토큰(파이프라인 design_system, ADR-0008)과
> *앱 자체*의 브랜드 토큰(본 ADR)은 다르다. 본 ADR 은 앱 자체 UI 만 해당.

## 결정
- **색 팔레트**(warm neutral): 배경 `#FAF9F5`/`#F5F4EE`, 표면 `#FFFFFF`/`#F0EEE6`,
  보더 `#E5E2D8`/`#DAD6CB`, 텍스트 `#1F1E1D`/`#6B6760`, 강조 `#C96442`(clay/coral),
  success `#5C8A4A`, danger `#B5524A`, warn `#B8893A`. CSS 변수 + Tailwind extend 로 이원화.
- **타이포**: 본문/디스플레이 = GmarketSans(한국어), 코드/데이터 = JetBrains Mono.
  design.html 의 `.font-serif` 는 실제 세리프가 아니라 GmarketSans 볼드(디스플레이용)로 그대로 유지.
- **질감/애니메이션**: paper(dot grid), skeleton shimmer, fade-up, pulse-dot, tab-btn 언더라인,
  diff-add/diff-rm, 커스텀 스크롤바. globals.css 에 포팅.
- **라우팅**: `/` = 랜딩(design.html), `/studio` = 대화형 스튜디오(기존 app 이동).

## 이유와 대안
- design.html 채택: 브리프가 지시한 감성을 이미 시각화한 진실 원천. 디자이너 핸드오프 전
  개발 단위에서 그대로 적용해 일관성 확보.
- CSS 변수 + Tailwind extend 병용: shadcn 호환(CSS 변수) + 유틸리티(Tailwind) 동시 만족.
- 폰트 CDN(GmarketSans): webfontworld CDN `<link>`. next/font self-host 이관은 차후.

## 영향
- 앱 전역 look&feel 이 warm neutral/clay 로 전환. 기존 result-viewer/generation-form 도
  새 토큰을 자동 상속(Tailwind 클래스 유지, 색만 교체).
- 프론트 의존: GmarketSans/JetBrains Mono 폰트 로드(CDN/next/font).
- design.html 은 디자인 진실 원천으로 레포에 보존(참고용).
- 향후 스튜디오 뷰 디자인은 본 토큰 위에서 별도 구현.
