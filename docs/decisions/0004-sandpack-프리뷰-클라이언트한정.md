# 0004. Sandpack 기반 프리뷰, 클라이언트 컴포넌트 한정
- 상태: 채택
- 날짜: 2026-07-01

## 배경
생성 코드를 사용자가 즉시 시각화하려면 브라우저 내 렌더링이 필요하다. WebContainers는
Next.js 서버 런타임까지 에뮬레이션하지만 설정이 무겁고 제약이 많다.

## 결정
**Sandpack**(CodeSandbox)을 사용하고, 프리뷰 대상은 **클라이언트 컴포넌트**로 한정한다.
서버 컴포넌트 전용 패턴은 코드 탭에서만 보여주고 프리뷰하지 않는다.

## 이유와 대안
- Sandpack은 설정 단순, 번들링·HMR 내장, Next.js client 컴포넌트 렌더에 충분.
- WebContainers 대안: 서버 런타임 지원하나 무겁고 복잡도 높음. MVP 오버스펙.
- 대안 iframe + 자체 빌드 서버: 인프라 비용 큼.

## 영향
- 생성 코드는 프리뷰 가능하도록 client 컴포넌트로 우선 설계해야 함( `'use client'`).
- 서버 전용 로직(data fetching 등)은 프리뷰에서 mock 데이터로 대체 표시.
- 향후 WebContainers 도입 시 확장 여지 열어둠.
