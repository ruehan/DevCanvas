# 0003. 계정/결제 제외, 익명 세션으로 MVP 출시
- 상태: 채택
- 날짜: 2026-07-01

## 배경
기획안 수익 모델(15장)은 무료/Pro/팀/기업 등급을 정의한다. 그러나 MVP 단계에서 계정·과금
인프라를 먼저 구축하면 핵심 가치(생성 품질)에 리소스가 분산된다.

## 결정
MVP는 **계정 없이 익명 세션**으로 운영한다. 프로젝트/히스토리는 세션 단위로 저장한다.
과금·팀 기능은 Phase 2+로 연기.

## 이유와 대안
- 핵심 가치 검증(생성 품질·코드 실사용)이 우선. 계정은 지연 원인일 뿐 가치 증명에 불필요.
- 익명 세션으로 사용자 진입 장벽 최소화 → 초기 피드백 확보 용이.
- 대안(최소 계정): 히스토리 저장엔 유리하나, OAuth/이메일 인프라 비용 발생.

## 영향
- 세션 만료 시 데이터 소실 가능 → 안내 명확화 필요.
- Phase 2 마이그레이션 시 세션→계정 매핑 설계 필요.
- 생성 횟수 제한(월 10회 등)은 IP/세션 기반 임시 적용.
