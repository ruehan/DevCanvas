# 0013. 리뷰 에이전트는 규칙 기반 린트 체크
- 상태: 채택
- 날짜: 2026-07-08

## 배경
review_agent 는 생성된 UI/코드의 품질(접근성·반응형·상태 누락·일관성)을 검사한다(기획안 F·17.4).
LLM 리뷰는 유연하나 비결정·누락 위험. 결정적 린트 체크가 오히려 "반드시 잡아야 할 것"을
보장한다. design/ui/ux/code(0008/0010/0011/0012)가 모두 규칙 기반이므로, 그 산출 코드에 대한
리뷰도 규칙 기반이 자연스럽고 회귀로 즉시 작동한다.

## 결정
- review_agent 는 **규칙 기반**으로 동작. `pipeline/review/checks.py` 의 체크 함수들이
  CodeGeneration.files 를 순회하며 패턴 매칭으로 ReviewFinding 을 생산.
- `pipeline/review/reviewer.py` 의 `run_review(code)` 가 체크들을 집계해 ReviewReport 반환.
- 체크 항목(MVP):
  - 상태 누락(P1): page.tsx 에 상태 분기가 TODO 로 남아있거나 아예 없음.
  - any 타입(P1): `: any` 사용.
  - 접근성(P2): page.tsx 에 aria-* 속성 없음.
  - 컴포넌트 미구현(P2): 컴포넌트 스텁에 TODO.
  - mock 데이터(P2): page 가 mock-data import (실제 API 교체 필요).
- LLM 리뷰(맥락적 판단·UX 흐름 등)는 향후 확장. 현재 llm.generate 를 사용하지 않는다.

## 이유와 대안
- 규칙 기반 린트: 결정적·즉시 회귀 적용·누락 없음. "반드시 잡아야 할 것" 보장.
- 대안(LLM 리뷰): 풍부하나 비결정·비용. MVP 에서는 린트 먼저, 향후 LLM 이 맥락 판단 추가(하이브리드).

## 영향
- review_agent 가 LLM 포트를 사용하지 않음 → 규칙 기반 파이프라인 구간 추가.
  이제 requirement만 LLM/더미, 나머지 6에이전트(ux/design/ui/code/review/handoff) 중
  review·handoff·code·ui·design·ux 가 규칙 기반(handoff는 다음 작업에서 정제).
- GenerationResult.review 가 코드 품질에 대한 구체적·재현 가능한 결과를 가짐.
- 체크는 패턴 매칭 기반이라 **위양성(오탐)** 도 가능하고 **위음성(주석/식별자로 우회)**
  도 가능. 향후 AST 기반 정제로 보강.
