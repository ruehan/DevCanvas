# 0016. 핸드오프 에이전트는 규칙 기반(코드에서 유도)
- 상태: 채택
- 날짜: 2026-07-08

## 배경
ADR-0009 로 디자인 토큰 산출을 파이프라인 코드에 연결했으나, handoff_agent 가 병합 전
`code`(CodeGeneration)를 받아 정적 더미 fixture 를 반환했기 때문에, file_tree·install_commands·
guide_md 가 실제 코드(페이지·컴포넌트 스텁·토큰 파일)를 반영하지 못하는 갭이 있었다(ADR-0009 영향).

## 결정
- handoff_agent 를 **규칙 기반**으로 전환. `pipeline/handoff/builder.py` 의
  `build_handoff(code, review)` 가 실제 코드에서 HandoffDoc 을 유도:
  - `file_tree`: code 파일 경로 정렬(토큰 파일 포함).
  - `install_commands`: 컴포넌트 스텁에서 알려진 의존(Tabs→@radix-ui/react-tabs,
    DataTable→@tanstack/react-table, Chart→recharts)을 검출해 `pnpm add` 목록 생성.
  - `todos`: review P1 findings + 표준(mock→API 교체, 컴포넌트 스텁 구현).
  - `guide_md`: 파일 구조·설치·TODO·P1 리뷰를 묶은 마크다운.
- LLM 정제(자연어 가이드)는 향후 확장. 현재 llm.generate 미사용.

## 이유와 대안
- 규칙 기반 유도: 실제 코드와 항상 정합(ADR-0009 갭 해소), 결정적·테스트 용이.
- 대안(LLM 가이드): 풍부하나 코드와 어긋날 위험. MVP 에서는 정합성 우선.

## 영향
- 이제 7에이전트 전부(requirement만 LLM/더미, 나머지 6은 규칙 기반)가 규칙 기반이다.
  핸드오프가 ADR-0009 토큰 파일·code_generator 산출을 모두 반영.
- **오케스트레이터 순서**: 토큰 병합을 `handoff_agent` 호출 **이전**에 수행한다. 그래야
  file_tree/guide_md 가 토큰 파일 5종을 포함한다. `review_agent` 는 토큰(결정적 산출)을
  린트할 필요 없으므로 병합 전 code 를 그대로 받는다. (리뷰 P1 반영)
- ADR-0009 의 handoff 갭(file_tree/install/guide 토큰 미반영) 해소.
- install 검출 매핑은 외부 의존만(DataTable→tanstack, Chart→recharts). shadcn/ui 기본
  제공 컴포넌트(Tabs/Card 등)는 제외 — `pnpm add @radix-ui/...` 로 오안내하지 않는다.
