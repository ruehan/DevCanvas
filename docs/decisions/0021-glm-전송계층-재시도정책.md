# 0021. GLM 전송 계층 재시도 정책

- 상태: 채택
- 날짜: 2026-07-09

## 배경
ADR-0007 은 재시도 정책을 "실 GLM 응답 특성(위반 빈도) 관측 후 별도 ADR" 로 이월했다.
`api/scripts/glm_observe.py` 로 실 GLM(z.ai `glm-4.7-flash`)을 관측했다:

- **requirement (RequirementSpec, 작은 스키마)**: 9/9 성공(100%). 스키마 위반·JSON 오류 0.
  엔티티/화면 산출도 프롬프트별 합당.
- **edit (GenerationResult, 거대 중첩 스키마)**: 초회 관측 0/3 성공 — **전부 전송 계층 오류**
  (`ReadTimeout` 1, `429 Too Many Requests` 2). **스키마 위반은 관측되지 않음.**

즉 실패 원인이 "응답 품질(스키마 위반)"이 아니라 "전송 계층(레이트리밋·타임아웃)"이었다.

재시도 도입 후 재관측(requirement 3/3 유지): **edit 는 재시도 후에도 429 지속 실패**.
무료 티어(~1 QPS)에서 거대 edit 요청은 짧은 지수 백오프(1s/2s)로 회복되지 않았다.
→ 재시도는 "일시적 blip" 흡수용으로 유효하나, **무료 티어 edit 실패의 근본 원인은
쿼터 대비 과대한 프롬프트**임이 확인됨(재시도로 해결되는 문제가 아님).

## 결정
1. **재시도 대상 = 전송 계층 일시 오류만.** `429`, `5xx(500/502/503/504)`, 타임아웃/네트워크
   오류(`httpx.TimeoutException`/`TransportError`)에 한해 지수 백오프로 재시도한다.
   - 기본 `glm_max_retries=2`(총 3회 시도), `glm_retry_base_delay=1.0s`,
     대기 = `base * 2**attempt`(1s, 2s, …).
2. **JSON 파싱/스키마 위반은 재시도하지 않는다.** 응답이 도착했으나 형식이 틀린 경우는
   같은 프롬프트 재호출 시 결정적으로 반복될 가능성이 높다 → 즉시 `GenerationError`.
   (관측상 위반 빈도 0 이므로 "오류 피드백 재프롬프트"는 근거 부족 — 도입 보류.)
3. **타임아웃 상향**: 기본 60s → `glm_timeout=120.0s`. 편집 턴 프롬프트가 거대해 60s 초과.
4. 재시도/타임아웃 파라미터는 전부 `Settings`(env `DEVCANVAS_GLM_*`)로 설정 가능.

## 이유와 대안
- **전송 계층만 재시도**: 관측 근거 직결. 위반이 0 인데 스키마 재프롬프트를 넣는 것은
  YAGNI 위반. 전송 오류는 재시도로 실제 회복 가능(일시적).
- **지수 백오프**: 429(레이트리밋, 특히 무료 티어 ~1 QPS)에 선형 재시도는 다시 429 유발.
- **대안(스키마 피드백 재시도)**: 위반이 관측되면 그때 별도 ADR. 지금은 데이터 없음 → 보류.
- **대안(타임아웃 유지 + 프롬프트 축약)**: 프롬프트 축약이 더 큰 레버지만 별도 작업(아래 영향).
  당장은 타임아웃 상향으로 관측된 실패를 직접 해소.

## 영향
- `GLMAdapter._fetch_content` 가 재시도 루프 담당. `generate` 는 프롬프트 구성 → fetch →
  파싱/검증 순서 유지. `sleep` 주입 가능(테스트에서 실제 대기 없이 백오프 검증).
- `Settings` 에 `glm_timeout/glm_max_retries/glm_retry_base_delay` 추가.
- **후속(별개·필수)**: `edit_agent` 의 프롬프트 비대(=`_build_prompt` 가 GenerationResult 전체
  JSON 스키마 + 결과 덤프 주입) 가 timeout·429·비용의 근본 원인. 재관측상 재시도만으로
  무료 티어 edit 이 통과하지 못함 → **프롬프트 축약(전체 스키마 대신 필수 필드/부분 수정)
  또는 캐싱**이 실질 해결책. 편집 사이클에서 별도 ADR 로 다룬다.
- 재시도 범위(한계 명시): 전송 계층 일시 오류만 흡수. 무료 티어의 지속적 429(쿼터 초과)는
  재시도 대상이나 짧은 백오프로는 회복 실패 관측됨 — 유료 티어/프롬프트 축약 전제 필요.
- 이 ADR 의 검증 상태: 재시도·백오프·타임아웃 로직은 **단위 테스트로 검증**(주입된
  timeout/429 시퀀스). 무료 티어 실 edit 통과는 프롬프트 축약(후속) 이후 재확인.
