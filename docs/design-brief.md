# DevCanvas 디자인 브리프 (디자이너 핸드오프) — v2

- 목적: DevCanvas 웹 앱 UI를 디자인할 디자이너에게 컨텍스트·방향·제약·산출물 전달.
- 작성일: 2026-07-09 (v2: 페이지 범위를 랜딩 + 대화형 스튜디오로 재집중)
- 상태: 디자인 착수 전

> 혼자 읽고 전부 이해할 수 있게 썼습니다. 막히면 언제든 질문해 주세요.

---

## 1. 제품 한 줄

**기획을 프롬프트로 입력하면 화면 구조·상태별 UI·디자인 토큰·복사 가능한 React 코드까지 만들어주고, 대화로 계속 다듬을 수 있는 개발자용 AI 디자인 스튜디오.**

- 핵심 가치: "예쁜 이미지"가 아니라 **"당장 붙여 쓸 수 있는 결과물"**.
- 차별점: 한 화면이 아니라 **전체 상태(loading/empty/error/permission/mobile)**까지 생성하고, **대화로 반복 수정** 가능.

## 2. 타겟 사용자 · 톤

- 1차: 프론트엔드/풀스택/1인 개발자. 디자인 리소스 부족, 빠른 MVP/관리자 페이지 필요.
- 톤: **전문적·차분·지적인 도구**. "신뢰할 수 있는 개발 파트너". 화려하거나 장난스럽지 않게.

## 3. 현재 상태 (참고 only)

지금은 **폼 + 7탭 결과 뷰**로 기능만 동작합니다. **이번 디자인에서 대화형 스튜디오로 전환**합니다(아래 5.2). 참고용 현재 파일:
- `web/src/app/page.tsx`, `web/src/features/generate-ui/`, `web/src/widgets/result-viewer/`

> DevCanvas가 **생성하는 결과물(토큰/코드)**과 DevCanvas **앱 자체 UI**는 다릅니다. 이 브리프는 **앱 자체 UI**를 다룹니다.

## 4. 디자인 방향: Claude Design 류 감성

레퍼런스: **Anthropic / Claude.ai / Claude Code** 의 디자인 언어. "지적인 개발 파트너" 정체성과 가장 잘 맞습니다.

### 4.1 핵심 무드
- **따뜻한 중립(Warm neutral)** — 차가운 SaaS 그레이가 아닌 종이/크림/흙감.
- **에디토리얼(Editorial)** — 문서/책처럼 차분한 위계. 세리프 타이포 힌트.
- **절제(Restrained)** — 그라데이션·글래스모피즘·네온·과한 그림자 없음. 여백이 넓고 선이 얇음.
- **대화형 도구(Agentic tool)** — Claude Code처럼 **대화와 산출물이 양쪽에 공존**하고, 작업 단계가 투명하게 드러남.
- **신뢰(Calm)** — 자극적 고대사 회피, 정보 위계로 신뢰.

### 4.2 색상 팔레트 제안 (시작점)
| 역할 | 제안값(예시) | 비고 |
|---|---|---|
| Background | `#FAF9F5` / `#F5F4EE` | 종이 같은 따뜻한 아이보리 |
| Surface (패널/카드) | `#FFFFFF` / `#F0EEE6` | 배경과 미세 대비 |
| Border | `#E5E2D8` / `#DAD6CB` | 낮은 대비, 얇은 선 |
| Text | `#1F1E1D` / `#2D2B27` | 검정 아님, 따뜻한 짙은 갈흑 |
| Text muted | `#6B6760` | 보조 텍스트 |
| Accent (강조) | `#C96442` / `#D97757` (clay/coral) | CTA·활성. 과용 금지 |
| Code/mono bg | `#F0EEE6` 틴트 | 코드/대화 명령 배경 |
| Success | `#5C8A4A` (따뜻한 세이지) | pure green 회피 |
| Danger | `#B5524A` (따뜻한 레드) | pure red 회피 |

- **라이트 우선**, 다크모드는 후속(따뜻한 다크 `#1F1E1D` 류 방향성만 메모).
- 드롭섀도 거의 사용 금지. 필요 시 극히 얕게.

### 4.3 타이포그래피
- **헤딩/디스플레이**: 세리프 계열(Source Serif, Lora, Tiempos 등) — 에디토리얼 무드.
- **본문/UI**: 산세리프(Inter / **Pretendard** 한국어 필수).
- **코드/명령/데이터**: 모노스페이스(JetBrains Mono / IBM Plex Mono / ui-monospace).
- 위계 예시: H1 28–32 / H2 20–22 / 본문 14–15 / 코드 13 / 캡션 12. 행간 본문 1.5–1.6.

### 4.4 형태·간격
- 모서리: 카드 `6–10px`, 버튼 `6–8px`. 16px 이상 과한 라운드 금지.
- 간격 4의 배수(4/8/12/16/24/32).
- 보더 1px 낮은 대비. 두꺼운 보더/그림자 대신 보더+여백으로 그룹핑.
- 호버: 미세한 배경 틴트 변화 정도.

### 4.5 절대 피할 것 (AI 테이스티 회피 — 기획안 18.5)
- 보라/파랑 그라데이션, 글래스 카드, 네온 글로우, 남발하는 2xl~3xl 라운드, 의미 없는 이모지/아이콘, 뻔한 SaaS 카드 3개 히어로, 모든 것에 드롭섀도.

## 5. 설계 대상 페이지 (★ 이번 범위)

범위는 **두 페이지**로 집중합니다.

### 5.1 랜딩 페이지 (`/`)
목적: 제품 가치 전달 + "시작하기"로 스튜디오 진입.

구성(제안):
- **히어로**: 가치 문장 + 한 줄 부가설명 + "시작하기" CTA.
  - 문장 후보: *"디자이너 없이, 기획만으로 구현 가능한 UI와 코드까지."*
  - 톤: 차분·에디토리얼. 자극 마케팅 회피.
- **데모/샘플**: 실제 생성 결과(예: 대시보드)의 정적/애니메이션 스냅샷 1개로 "무엇이 나오는지" 즉시 보여주기.
- **핵심 특징 3개**: (1) 프롬프트 → 전체 화면 구조, (2) 상태별 UI(State Matrix), (3) 코드까지 복사.
- **사용 흐름 3단계**: 프롬프트 입력 → 결과 보기/고치기 → 내보내기.
- 푸터(최소).

모바일 대응 필수.

### 5.2 디자인 스튜디오 (`/studio` 또는 `/`) ★ 핵심
**Claude Code**처럼 **프롬프트로 디자인을 생성하고, 후속 대화로 수정**하는 페이지.

#### UX 패러다임
- **대화(Conversation)** + **산출물(Artifact)** 이 양쪽에 공존.
- 사용자는 자연어 프롬프트로 요청 → 에이전트가 생성/수정 → 우측 산출물이 실시간 갱신 → 다시 대화로 정제.
- 단순 폼 제출이 아니라 **멀티턴 대화**. 각 턴마다 산출물이 진화.

#### 레이아웃 (데스크톱 기본)
좌-우 분할(비율 제안: 좌 38~42% / 우 나머지, 조정 환영):

**좌측 — 대화 패널**
- **대화 스레드**: 사용자 메시지(프롬프트) + 에이전트 응답/작업 단계.
  - 에이전트 응답은 "무엇을 했는지" 요약(예: *"고객 목록 화면을 추가하고, 빈 상태 CTA를 넣었어요."*).
  - **작업 단계 노출**: Claude Code처럼 에이전트가 여러 단계(요구사항 분석 → UX 설계 → 디자인 시스템 → 코드 생성 → 리뷰)를 거치는 과정을 접기/펼치기로 표시(투명성).
  - 변경 사항은 **diff 느낌**(무엇이 바뀌었는지)으로 가볍게 표현.
- **입력 박스(하단 sticky)**: 멀티라인 프롬프트 입력.
  - placeholder 예: *"고객 관리 관리자 페이지를 만들어줘. 결제 상태 필터와 상세 보기 포함."*
  - 후속 수정 예: *"버튼을 조금 더 둥글게"*, *"모바일에서 카드형으로 바꿔"*, *"다크 톤으로"*, *"고객 테이블에 계약일 추가"*.
  - 전송 버튼, 엔터/Shift+엔터 가이드, 예시 프롬프트 칩(빈 상태).
- (선택) 상단: 프로젝트명, 모델 표시, 새 대화.

**우측 — 산출물 캔버스 (라이브 아티팩트)**
대화가 진행될수록 채워지는 **실시간 결과물**. 모드(탭/분할)로 전환:
- **Preview** ★: 생성된 화면의 실시간 프리뷰(Sandpack). 화면 셀렉터 + **상태 토글**(default/loading/empty/error)로 State Matrix 체감.
- **Code**: 파일 트리 + 모노스페이스 코드 패널 + 복사 버튼.
- **Design System**: 토큰(색/간격/반경/타이포/그림자) — 색상 칩은 실제 색으로.
- **States**: 화면×상태 매트릭스(킬러 기능 13.1).
- **Review**: P1/P2 리뷰 항목.
- **Export**: 파일 트리 + 설치 명령 + TODO + 가이드 + ZIP 다운로드.

> 탭 구조는 현재 구현과 동일하되, **대화에 의해 갱신**되는 것이 핵심. 탭 vs 분할(Preview 고정 + 코드 사이드) 디자이너 제안 환영.

#### 상태 (모두 필요)
- **빈 상태(신규 대화)**: 좌측엔 예시 프롬프트 칩 + 가이드, 우측엔 "프롬프트로 시작하세요" 플레이스홀더.
- **생성 중(loading)**: 에이전트 단계 진행 표시(어디쯤인지), 우측은 스켈레톤/자리표시자. 과한 스피너 금지.
- **에러**: 명확한 메시지 + 재시도.
- **대화 길어짐**: 좌측 스크롤, 최신 응답으로 자동 포커스. 산출물은 항상 최신 상태 유지.

#### 모바일
- 세로 스택: 대화가 메인, 산출물은 토글/하단시트로 전환. 정보 밀도 조정.

#### 인터랙션 디테일
- 산출물 갱신 시 **부드러운 전환**(점프/플리커 회피), 변경된 부분 하이라이트(가능 시).
- 코드 복사 버튼, 화면 이미지 내보내기(후속).
- 키보드 친화: 입력 포커스, 단축키(전송/중단).

## 6. 산출물 (디자이너에게 요청)

1. **디자인 토큰 정의** (Figma Variables 또는 JSON): 색/타이포/간격/반경/그림자. → Tailwind `theme.extend` + shadcn CSS 변수 호환.
2. **핵심 화면 Hi-Fi**:
   - 5.1 랜딩(히어로 + 특징 + 흐름)
   - 5.2 스튜디오 **3 상태**: (a) 빈 대화(신규), (b) 생성 중(진행 단계), (c) 결과 생성 후(대화 + 산출물 채워짐).
3. **컴포넌트 스펙**: 버튼(variants), 입력/textarea(멀티라인), 채팅 버블(사용자/에이전트), 작업단계(접기/펼치기), 탭, 파일 트리, 코드 블록(복사 버튼), 토큰 칩/표, 칩/badge, 모달, 토스트, 스켈레톤. → shadcn 매핑.
4. **상태 디자인**: 빈/로딩(진행)/에러.
5. **반응형**: 데스크톱(1280+) 우선, 태블릿(768), 모바일(375).
6. **(선택) 다크모드 방향성** 메모, **타이포 페어링**(한글 포함).

## 7. 기술 제약 (구현 호환)

- **Tailwind + shadcn/ui**. 토큰은 shadcn CSS 변수(`--background`, `--foreground`, `--primary` …) 호환 권장.
- Next.js App Router. 코드는 모노스페이스 + (향후) 신택스하이라이트.
- 아이콘: lucide-react 선호. 커스텀 일러스트 최소.
- 폰트: `next/font` 로 로드(셀프호스팅 권장).

## 8. 참고 레퍼런스

- **Anthropic / Claude.ai / Claude Code** — 따뜻한 중립 + 에디토리얼 + 대화·산출물 공존 UI. ★ 주 레퍼런스.
- Vercel v0(대화형 생성 부분), Linear(정보 밀도·절제), Vercel/Stripe Docs(코드 패널·위계).
- (회피) 일반적 AI 앱 빌더의 자극적 히어로/그라데이션.

## 9. 성공 기준 (디자인 관점)

- "Claude 류의 차분하고 지적인 도구" 인상을 3초 안에 주는가.
- 스튜디오에서 **대화로 생성→수정→확인** 흐름이 막힘없이 한 화면에서 도는가.
- 산출물(코드/토큰/상태)을 **읽기 편한 밀도와 위계**인가.
- 토큰이 Tailwind/shadcn 으로 그대로 옮겨지는가.

## 10. 협업 메모

- 1차 라운드: 토큰 초안 + 핵심 Hi-Fi(랜딩 1 + 스튜디오 3상태)로 리뷰.
- 모호하면 합리적 가정 후 제안해 주세요.
- 토큰 확정 시 백엔드가 **생성하는 결과물 기본 톤**(tone 프리셋)과의 관계도 논의(앱 UI 톤 ≠ 생성 결과물 톤).
